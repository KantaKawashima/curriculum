ウォーターフォール型開発
開発手順を1つずつ確認しながら工程を進めていく手法のこと。
開発担当や責任者、クライアントが各工程の成果物を確認し合意を得てから進めていく。

アジャイル型開発
リリースの早さを意識した開発手法。相殺する機能を最低限の状態にする。
開発メンバーが優先順位をつけ、短い期間での納品を目指す。
優先順位を決めるために毎日ミーティングを行い、チーム内で連携をとるスクラムという
手法が取られる。

マイグレーション
ソフトウェアやシステム、データなどを別の環境に移転したり、
新しい環境に切り替えたりすることを意味する


デグレーション
バージョンアップしたものが前のバージョンと比べ、
機能が落ちていること

デプロイ
開発した機能やサービスを利用できる状態にする作業

コンフリクト
複数のプログラムが同じリソースを利用しようとして競合状態になってしまうこと。

環境
開発環境（ローカル）
システムやソフトウェアを開発するために必要な機材やソフトウェアの組み合わせ。

検証環境（ステージング）
開発し終わったリリース予定の機能をテストするための環境。

本番環境
ユーザーが実際にシステムを使う環境のこと。

設計書／仕様書
何を作るのか説明した資料
どうやって作るのか説明した資料

要件定義
どうシステム化するのかを決めるのが要件定義

基本設計書
ソフトウェアの動きを外から見た際にどういう動きをするのか決めたもの

詳細設計書
基本設計で決められた動きをどうやって実現するか決めたもの

テスト
単体テスト（UT）
結合前の状態でシステムが動くかどうか確認するテスト

結合テスト（CT／IT）
それぞれのプログラムが結合後の状況で動作するかどうか確認するテスト

総合テスト（ST）
開発側の最終テストのこと

エビデンス／証跡プログラムやシステムの試験の結果や、
動作検証を確認したことを証拠として残すために作成する。

データパッチ
ソフトを構成するプログラムを更新するために差分をまとめたデータのこと。

Access
機器やソフトを利用可能な状態にすること。

マクロ
関連する操作や手順、命令などを一つにまとめ、必要に応じて呼び出すことができるようにする機能。

VBA
利用者が簡易なプログラムを記述して実行することで複雑な処理の自動化などを行なうことができるもの。
そのために用意されたプログラミング言語。

GAS（Google Apps Script）
Javascriptをベースにした言語。
グーグルのサービスと連携できる。

API
WEB API
Web上でのAPIのやり取りを可能にしたもの
(amazon,楽天）

REST API
本的に「RESTの原則」に従って実装されているWebシステムのHTTPでの
呼び出しインターフェースのことを指す。

SOAP
異なるコンピュータ上で動作するプログラム同士がネットワークを通じてメッセージを伝え合い、
連携して動作するための通信プロトコル（規約）の一つ


Ajax
サーバーとの通信を非同期で行いながら
インターフェースの構築を行う手法

XML
拡張可能なマークアップ言語。
分かりやすく、拡張性が高い。

JSON
JavaScriptのオブジェクト記法を用いたデータ交換フォーマット。

ステータスコード
HTTPステータスコード（HTTP Status Code）とも呼ばれる、
HTMLを表示する前にブラウザが受信する3桁の番号です。
HTMLが受信する前に受信することでブラウザで正しく処理できる。

クロスオリジン（Cross-Origin）／CORS（Cross-Origin Resource Sharing）
同一生成元ポリシーというポリシーによって設けられた制限を緩めるものです。
アクセスできる範囲を増やす。

バリデーション
対象がその仕様や文法などに照らして適切に記述・構築されているか否かを検証する

データ連携
Daemon
UNIX系OSでメインメモリ上に常駐して特定の機能を提供するプログラム。
利用者の操作とは無関係に実行されるプログラム。

CRON
デーモンの一種。利用者の設定したスケジュールに従って指定されたプログラムを定期的に起動してくれる。

バッチ
複数のプログラムからなる作業において、あらかじめ一連の手順を登録しておき、
まとめて連続的に実行する方式。

ログ
起こった出来事についての情報などを一定の形式で時系列に記録・蓄積したデータのこと。

CSV
カンマで値を区切ったファイル。表計算ソフトやデータベースソフトの管理に適したファイル。

TSV
データベースにおけるデータをテキストファイルとして保存する形式。

プルリクエスト（マージリクエスト）
開発者のローカルリポジトリで加えた変更を、他の開発者に知らせ、レビューしてもらう機能。
コードミスやエラーに気づくことができ、品質を高めることができる。
