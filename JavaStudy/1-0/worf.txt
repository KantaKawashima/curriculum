5,SQL
・SQL

データ管理システム大容量動作を行える。

MySQLから派生したデータベースシステム

性能や堅牢性を高めるための独自機能を追加。

読み取りの一貫性（検索した段階で更新されたデータも読み取る）

PostgreSQL
オープンソフトウェアのためコストが安い。
パラレルクエリという処理要求の実行を並列化させるために
複数のCPUを活用して性能向上を図るシステムがある。

SQLite
ファイルベースのデータベースエンジン。
ロックが柔軟ではないのが特徴。
ロックする範囲をデータベース全体でしか制御できない。
開発時に使うことが多いが運用はMySQLやPostgreSQLを使われることが多い。

SQL Server
テーブル同士を関連付けて管理するデータベースのため直感的に操作ができる。
SSMSという総合管理ツールを使いデータベースの作成や更新を行うことができる。
無料版と有料版がある。


・CRUD
Create、Read、Update、Deleteの四つの機能。
ユーザーインターフェースが備えるべき機能を指す。
・トランザクション
お互いに関連する複数の処理をまとめた処理単位として扱うこと。

コミット
トランザクションで結果を確定させること。
結果を反映させること。
Gitupなどの変更履歴の保存や管理を行うシステムで結果を反映させること。

ロールバック
トランザクション処理の中で結果をなかったことにすること。

・排他制御（排他ロック）
あるトランザクションがアクセスしている際に他のトランザクションからはアクセスできないようにすること。
データに不具合が起きないようにする機能。

楽観ロック
基本的にめったに他者との同時更新は起きないだろうという前提の排他制御。
データそのものはロックをせず。更新時のデータがデータを取得時と同じであることを確認してから
更新をする。

悲観ロック
他者が同じデータに頻繁に変更を行うという前提の排他制御。
更新対象のデータを取得する際にロックを掛けることで他のトランザクションから完全に
更新できないようにする。
同一業務を同時に複数人で行う場合に使われる。



実行計画

スロークエリ
MySQL で出力できるログ。

これにより、アプリケーション構築時や本番運用時にパフォーマンス
のボトルネック（全体の能力や速度に影響し散るもの）となっている SQL を発見するのに大いに役に立つ。

クエリログ
MYSQLで実行されたすべてのSQLクエリをすべて記憶する。
エラーが起きた際にどのようなSQLが実行されたか確認できる。
